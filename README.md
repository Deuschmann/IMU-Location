##  简介

*   **极高精度**: 累积总距离误差率仅为 **0.88%**（在 235m 测试路径上仅偏移 2.08m）。
*   **两阶段架构 (2-Stage)**:
    *   **Stage 1 (MLP)**: 局部特征提取与初步去噪，通过 Softplus 激活函数输出物理意义明确的基准步长。
    *   **Stage 2 (GNN)**: 利用 GATv2Conv 构建 K-Hop 因果图，对步长进行动态乘法缩放校准。
*   **严格实时因果性**: 所有特征工程（包括基于 EMA 的重力估计和滑动窗口方差）均不依赖“未来数据”，确保模型可直接部署于移动端实时场景。
*   **多维度量化评估**: 提供从步长误差、航向角漂移到最终 2D 轨迹 ATE (平均定位误差) 的全方位评估。

##  模型架构

### 1. 特征工程 (Real-time Feature Extraction)
模型从原始加速计和陀螺仪数据中提取 12 维特征，包括：
*   通过 EMA（一阶低通滤波）实时估计重力向量。
*   坐标轴对齐：提取垂直加速计分量 ($acc\_v$) 和水平投影分量 ($acc\_h$)。
*   滑动窗口统计：捕获运动剧烈程度的方差特征。

### 2. 网络设计
*   **局部去噪层 (Stage1MLP)**: 处理 $11 \times 12$ 的滑动窗口输入，提取高维运动特征。
*   **时间关联层 (Stage2GNN)**: 构建时间序列图，节点间通过注意力机制传递时空信息，预测步长的缩放系数 $\exp(scale)$。

##  实验结果 (Benchmark)

在标准数据集测试中，模型表现如下：

### 1. 距离估计性能 (Model Core)
| 指标 | 数值 |
| :--- | :--- |
| **逐帧平均绝对误差 (MAE)** | 0.0058 m/frame |
| **逐帧均方根误差 (RMSE)** | 0.0080 m/frame |
| **累积总距离误差 (235m)** | **2.08 m (0.88%)** |

### 2. 2D 定位误差 (Trajectory Error)
| 测试场景 | 平均定位误差 (ATE) | 90% 分位数 (CEP90) |
| :--- | :--- | :--- |
| **模型上限 (Perfect Yaw)** | **1.39 m** | **1.89 m** |
| **工程现实 (Raw IMU Yaw)** | 6.27 m | 7.38 m |

##  安装与使用

### 环境依赖
*   Python 3.8+
*   PyTorch
*   PyTorch Geometric (PyG)
*   Pandas, Numpy, Matplotlib

### 运行
1. 准备数据集：OXIOD。
2. 执行主程序进行训练与评估：
```bash
python model_predict.py
```
